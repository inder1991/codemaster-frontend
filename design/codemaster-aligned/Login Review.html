Design review · Sprint 14

Login page — comprehensive UX/UI audit

A frank, head-of-design pass on /login ahead of the 5,000-engineer pilot. Findings are ranked by user-impact and ordered for the smallest viable fix first.

Reviewer

Head of Design

2026-05-10

Overall

B−

Functional and on-brand, but emotionally inert. The page works for engineers who already know what codemaster is, and fails newcomers, accessibility audits, and anyone landing here from a session timeout. The right-side product preview is the strongest thing on the page; the form column is technically correct but underexplained, and the error vocabulary contains at least one piece of copy that breaks user trust.

12 findings · 1 blocker 4 issues 5 suggestions 2 nits

What's working

Wins
  • Right-column product preview. The single, polished GitHub comment is the most persuasive thing on the page. It tells the story of what codemaster does in three seconds — diff hunk + suggested change + citation footnote — and quietly proves the trust-traceability invariant from PRODUCT.md.
  • Brand identity is consistent. Amber accent, dot-grid texture, and ambient glow read as one product. Dark mode is properly themed end-to-end (no white flashes on field focus).
  • Caps-lock detector in the password field is a small, specific care signal that engineers notice.
  • SSO is the visual primary, directory is the visual secondary. Hierarchy matches actual usage — most pilot users will be on Okta.

Blocker

L-01 Blocker The ldap_unreachable error tells unauthorized users how to escalate

Current copy: "Auth service temporarily unavailable. The local super-admin path is still working, try a super-admin credential if you have one."

This is a security disclosure on a public-facing form. Anyone — including someone testing credentials they shouldn't have — learns (a) that a fallback admin path exists, and (b) that it accepts shared super-admin credentials. The internal recovery hint belongs in a runbook, not on a banner shown to every failed login attempt during an LDAP outage.

Fix Replace with "Auth service temporarily unavailable. Please try again in a few minutes, or contact your platform owner." The super-admin path can still be used; it just stops being advertised.

Issues

L-02 Issue "Welcome back" greets first-time pilot users with a lie

5,000 engineers across 60+ orgs are about to see this page for the first time. The H1 assumes prior visit and gives them no idea what they're signing into. There's no product positioning, no "what is this," no link to the docs hub.

The right-column preview compensates partially, but it's hidden below ~1024px and on the small percentage of users who land via "session-ended" the warmth is misplaced.

Fix Detect first-visit (no prior session cookie) and swap to "Sign in to codemaster" with a one-line subhead: "AI code review for the platform org — every finding cites a source." Keep "Welcome back" only when a prior session is detected.
L-03 Issue No "remember me" / persistent session control

Engineers will hit this page multiple times a day on the pilot. There's no checkbox to opt into a longer session, no indication of session length, and no way for a security-conscious user to opt out of long sessions on a shared machine. Auth-product convention covers both.

Fix Add a "Keep me signed in for 7 days" checkbox below the password field, default-on for SSO and default-off for directory. Show the resulting expiry as helper text.
L-04 Issue Password field has no "show password" affordance

Standard pattern; absence is conspicuous on a page that otherwise feels considered. Especially important here because the directory path is the fallback — when SSO fails and a user is hand-typing a long LDAP password under stress, they need to verify it.

Fix Add an eye-toggle in the password field's trailing slot, alongside the existing caps-lock pill. Persist the show/hide state for the lifetime of the form only.
L-05 Issue Right column hidden below 1024px with no replacement

On laptops at 15" in split-window mode and on iPads, the entire product-preview disappears. The user is left with a small centered form that gives them zero context for what codemaster is. The empty space is also genuinely ugly — the ambient glow + dot grid was tuned for the two-column layout.

Fix Below lg, collapse the preview to a compact one-line summary above the form: "AI code review · 3,074 repos · every finding cites a source" with a small severity-pill row. Re-tune the ambient glow for the single-column case.

Suggestions

L-06 Suggestion SSO button doesn't tell users where they're going

"Continue with Acme SSO" is generic. The earlier "routes through Okta" subtext was the right instinct — losing it to reduce clutter was probably the wrong trade. Engineers about to be redirected want to know which IdP, especially during an Okta-vs-Azure-AD migration.

Fix Bake the IdP into the button itself: "Continue with Okta" + the Okta wordmark glyph. Removes the subtitle and raises information density.
L-07 Suggestion "OR USE DIRECTORY" reads as second-class punishment

For platform-owners and during incident response, the directory path is the preferred path because it bypasses any IdP outage. The current language signals it as the lesser option.

Fix Soften to "Or sign in with your directory account", lowercase, no all-caps. Keep visual hierarchy the same; just stop calling it a fallback.
L-08 Suggestion No status link for "is codemaster down right now?"

Internal tools live or die by their status page. The right-column does the trust work for the product, but if the user is sitting on a failed sign-in, the natural next question is "is this on me or on the system?" There's no link to /status, even though the page exists in the app.

Fix In the footer, alongside the version, add "All systems operational" with a healthy dot, linked to /status. Pull the actual state from the status endpoint so it's never stale.
L-09 Suggestion "Contact your platform owner" link is a dead end

It's an href="#". The user clicks it expecting an email address, a Slack channel, or a runbook. Nothing happens. Worse, this link only appears under the form — not on the error banners themselves, where it would matter most.

Fix Wire it to the configured platform-owner contact (per-org, from the same config that drives the audit-log routing). Mirror the link inside the danger banners for no_role and locked.
L-10 Suggestion Loading / submitting state is missing

The "Sign in" button has no busy state. On a slow LDAP bind a user can double-submit, and the SSO redirect can take 1–2 seconds during which nothing acknowledges the click. Both are common sources of "did it work?" support tickets.

Fix Disable both auth buttons on submit and swap the label for an inline spinner + "Signing in…" / "Redirecting to Okta…". Re-enable on response.

Nits

L-11 Nit Bullet caret on the SSO chevron

The directory "Sign in" button has a right-arrow chevron; the SSO button doesn't. Inconsistent. SSO is the redirect that actually takes the user away from the page, so it's the one that semantically deserves the chevron.

Fix Move the chevron to the SSO button. Keep the directory submit button clean.
L-12 Nit Footer copy is unintentionally adversarial

"Internal tool. Authorized employees only." is correct, but it's the only sentence on the page in a defensive tone. On a page that otherwise feels welcoming, it lands harder than intended.

Fix Swap to "For Acme employees only · Privacy & data handling →" with the second half linked to the data-handling note.

If you only fix three things

  1. L-01 — Rewrite the LDAP-unreachable banner before pilot. Security risk, fast fix.
  2. L-04 — Add show-password. Single highest-impact UX nit; users will notice its absence.
  3. L-05 — Make the right column responsive. Half your pilot users will see this page on a sub-1024px viewport.

Will people love this page?

Engineers won't love a login page — they'll love the product behind it. What this page needs to do is: (a) earn trust in the first three seconds, (b) get out of the way for repeat visits, (c) not make a bad day worse on the error path.

The current draft does (b) well, half-does (a) thanks to the right-column preview, and outright fails (c) because the error vocabulary leaks internal information and the recovery paths are dead ends. Address the blocker and the four issues and this page goes from "competent" to "the kind of internal tool people quietly tell each other about."
